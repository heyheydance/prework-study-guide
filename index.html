HTML

    HTML

    • The head element contains information about the webpage.
    • The !DOCTYPE declaration tells the browser what document type it should expect when it is opened.
    • The charset element sets our character encoding.
    • The http-equiv element sets an HTTP header for content.
    • The viewport element sets the default view for our webpage.
    • The body element represents the visible content shown to the user.
    • The section element creates different sections for you to work in.
    • The style element defines the CSS styles associated with a website.
    • The base element defines the base URL for a webpage.
    • The link element connects an external resource to the HTML document.
    • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
    • The script element embeds client-side scripts in the HTML document.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • We can use inline CSS directly in whatever tag we want to manipulate.
  • We can create an internal style sheet using the style tag to hold all of our rules.
  • We can create an external style sheet and incorporate it with the link tag.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property
  • A margin property indicates how much space we want around the outside of an element
  • The padding property adds space around the content inside an element.
  • The box-shadow CSS property adds shadow effects around an element's frame.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: stage your changes for merging.
  • git commit -m "added code to HTML file": commit the changes.
  • We use the -m flag to associate a message with our commit.
  • git pull origin main: pull in the main branch.
  • It is a good habit to always pull the base branch into your feature branch before pushing up any code.
  • git push origin [working branch]: push all of the changes to the GitHub repository.
  • Create a pull request on GitHub from the working branch to the main branch.
  • Merge with the main branch to have all the code fully updated.

JavaScript

  • JavaScript is the tool that takes a static HTML/CSS webpage and makes it interactive.
  • A variable is simply a named container that allows us to store data in our code.
  • In JavaScript, to declare a variable, we use the var keyword.
  • We can use the console.log method to output a message to the web console by adding an argument.
  • Javascript files are linked within the body element.
  • We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
  • The code to be executed in a function is written inside the curly brackets {}.
